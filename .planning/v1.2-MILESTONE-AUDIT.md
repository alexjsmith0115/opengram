# v1.2 Milestone Audit — Gaps Found

## Summary

Phases 15, 16, 17 are internally clean. Phase 18 ships unit-verified rephrase card components but has a **critical hotkey-path wiring gap** (WIRE-01): `CheckCoordinator` routes all scheduler LLM results to `LLMPanelController` (v1.1 per-issue panel path) and never to `OverlayController`. `OverlayController.tryDispatchRephraseCard()` filters `.source == .llm` but always receives empty `llmIssues` — the rephrase card cannot fire in the real hotkey path under any flag combination.

All REPH-01..REPH-13 unit tests pass in isolation; none exercise the coordinator → overlay wire.

## Blockers

### WIRE-01: Coordinator LLM-suggestion routing (high)

- **Where:** `OpenGram/App/CheckCoordinator.swift:110` (Harper-only to OverlayController), `:141–161` (scheduler results → LLMPanelController exclusively)
- **Filter:** `OpenGram/SuggestionUI/Overlay/OverlayController.swift:355` (`llmInRange` always empty in hotkey path)
- **Effect:** 13 REPH requirements are unit-verified but unreachable end-to-end.
- **Fix:** After `scheduler.check()`, merge `schedulerSuggestions` (Harper + LLM) and call `overlayController.update()` with the merged set. Keep `LLMPanelController` path for legacy flag-off v1.1 behavior only, or make its invocation conditional on `!paragraphRephraseCardEnabled`.

## Tech Debt

- **UI-01:** Both feature flags (`llmParagraphRephraseCardEnabled`, `llmIncrementalCheckingEnabled`) are CLI-only. Could be intentional rollout gate — needs spec confirmation.
- **REQ-STALE-01:** 6 REQUIREMENTS.md entries (INCR-02, INCR-04, INCR-09, INCR-13, SET-08, SET-09) show "Pending" but are complete with tests.
- **DI duplication:** AppDelegate constructs two `UserDefaultsIncrementalConfig` instances; low severity, both read the same store.

## Deferred (→ Phase 19 UAT)

- 16-04 Task 5: live Notes/TextEdit flag behavior
- 17: live three-tab settings validation
- 18-08 Task 4: 12-step Notes.app rephrase card flow (**blocked by WIRE-01 — cannot UAT until fix lands**)

## Recommendation

**Do not complete the milestone until WIRE-01 is fixed.** The rephrase card is the headline v1.2 feature and ships unreachable. Create a decimal phase (e.g., 18.1) or append a gap-closure task:

1. Wire scheduler LLM Suggestions into `overlayController.update()` alongside Harper.
2. Gate `LLMPanelController` invocation on `!paragraphRephraseCardEnabled` to avoid double-panel when card is active.
3. Add an integration test: mock scheduler returns qualifying LLM suggestions → `overlayController.update()` → `tryDispatchRephraseCard` fires → card presented.
4. Update REQUIREMENTS.md to reflect the 6 stale entries.

UI-01 (flag toggles) can be deferred if rollout gate is intentional.
