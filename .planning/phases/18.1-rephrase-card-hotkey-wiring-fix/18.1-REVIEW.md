**Reviewed:** 2026-04-17

## Summary

WIRE-01 routing fix is structurally correct. LLM suggestions now reach `overlayController.update()` and `tryDispatchRephraseCard` will receive non-empty `llmInRange`. Flag gate preserves v1.1 parity. No security issues, no crashes, no index corruption.

Three warnings: (1) the update guard condition triggers a redundant card re-evaluation pass when LLM returns empty, (2) `UserDefaultsIncrementalConfig()` instantiated ad-hoc breaks the established DI discipline, (3) the flag-gate regression tests are tautologies — they test boolean algebra, not coordinator behavior, and cannot catch a regression in `CheckCoordinator.swift`.

One info item: the WIRE-01 integration test exercises the merge contract directly against `OverlayController` but bypasses `CheckCoordinator.handleHotkeyFired()`, leaving the coordinator's merge path untested by automation.

## Warnings

### WR-01: Redundant overlay update when LLM returns no suggestions

**File:** `OpenGram/App/CheckCoordinator.swift:169-172`

**Issue:** The guard condition `!llmSuggestions.isEmpty || !self.accumulatedSuggestions.isEmpty` fires even when `llmSuggestions` is empty (e.g., LLM ran but returned only `.harper`-sourced results, or nothing). In that case `merged == accumulatedSuggestions` — identical to what was already in the overlay after `checkTask`. This triggers a second `tryDispatchRephraseCard` evaluation with an empty `llmInRange`, which wastes the `WR-02` hash guard check and redundantly re-runs paragraph splitting and heuristic evaluation on every LLM completion, even cold ones.

```swift
// current — fires even when llmSuggestions is empty
if !llmSuggestions.isEmpty || !self.accumulatedSuggestions.isEmpty {
    let merged = self.accumulatedSuggestions + llmSuggestions
    self.overlayController.update(suggestions: merged, context: context)
}
```

**Fix:** Gate on LLM results actually being present:

```swift
if !llmSuggestions.isEmpty {
    let merged = self.accumulatedSuggestions + llmSuggestions
    self.overlayController.update(suggestions: merged, context: context)
}
```

The card can only dispatch when `llmInRange` is non-empty, so calling `update()` with Harper-only content yields no card. Skipping the call when `llmSuggestions` is empty is strictly equivalent for card dispatch and avoids the redundant evaluation.

### WR-02: `UserDefaultsIncrementalConfig()` instantiated ad-hoc — breaks DI discipline

**File:** `OpenGram/App/CheckCoordinator.swift:163`

**Issue:** `CheckCoordinator` has DI-injected `scheduler` (which itself holds an `IncrementalConfig`), yet the flag gate reads a freshly-constructed `UserDefaultsIncrementalConfig()` — a second `UserDefaults.standard` reader not visible to tests or the DI graph. This is structurally identical to the "AppDelegate DI duplication" CONTEXT.md calls out as low-severity tech debt. Since the CONTEXT explicitly chose "inline flag read — least-invasive," this is a known deviation, but it makes the flag gate untestable without `UserDefaults` injection.

```swift
// current — bypasses DI; untestable without real UserDefaults
let cardEnabled = UserDefaultsIncrementalConfig().paragraphRephraseCardEnabled
```

**Fix (minimum viable):** Expose `paragraphRephraseCardEnabled` via the `scheduler`'s injected config, or pass it through `CheckCoordinator`'s existing `scheduler` dependency:

```swift
// scheduler already holds an IncrementalConfig; read through it
// (requires IncrementalConfig to be accessible on LLMCheckScheduler, or
//  CheckCoordinator to hold its own injected IncrementalConfig instance)
let cardEnabled = incrementalConfig.paragraphRephraseCardEnabled
```

If the "no DI change" constraint must hold for 18.1, accept as tech debt and track alongside the existing AppDelegate duplication note. Do not copy this pattern further.

### WR-03: Flag-gate regression tests are tautologies — cannot catch coordinator regression

**File:** `OpenGramTests/SuggestionUITests/OverlayControllerFlagOffRegressionTests.swift:45-73`

**Issue:** `CheckCoordinatorFlagGatingTests.flagOn_panelSuppressed_branchLogic` and `flagOff_panelInvoked_branchLogic` both assert pure boolean algebra:

```swift
#expect(!config.paragraphRephraseCardEnabled == false)  // tests: !true == false
#expect(!config.paragraphRephraseCardEnabled == true)   // tests: !false == true
```

Neither test constructs a `CheckCoordinator`, calls `handleHotkeyFired()`, or verifies that `showLLMPanel` is invoked or suppressed. They will pass even if the entire `if !cardEnabled { self.showLLMPanel(...) }` block is deleted from `CheckCoordinator.swift`. They provide zero regression protection for WIRE-01's flag gate.

**Fix:** At minimum, add a comment explicitly stating these are compile-time guard assertions (not behavioral tests) and that behavioral coverage is deferred pending `LLMPanelController` protocol injection. Better: extract `LLMPanelController` behind a protocol so a spy can be injected, and assert invocation count per flag state.

```swift
// Minimal honest comment:
// NOTE: These tests verify boolean identity of the gate expression only.
// Behavioral coverage (spy on LLMPanelController) requires protocol
// injection — deferred to a future phase per 18.1 CONTEXT.md minimize-diff constraint.
```

If these are intentionally placeholder tests documenting the deferred work, name them `flagGate_branchExpression_compilesCorrectly` or similar so reviewers don't mistake them for behavioral assertions.

## Info

### IN-01: WIRE-01 integration test bypasses `CheckCoordinator` — merge path untested

**File:** `OpenGramTests/SuggestionUITests/OverlayControllerRephraseIntegrationTests.swift:59-108`

**Issue:** `hotkeyFired_schedulerLLMSuggestions_reachOverlayController` directly calls `overlayCtrl.update(suggestions: merged, context: ctx)` with a manually constructed `merged` array. It verifies that `OverlayController` stores LLM suggestions correctly, but does not invoke `CheckCoordinator.handleHotkeyFired()`. A regression that reverts the merge in `CheckCoordinator` (e.g., restoring the single `showLLMPanel()` line) would leave this test green.

The SUMMARY acknowledges this: "uses `overlayController.suggestions` as proxy — avoids requiring AX element in tests." Accepted trade-off for 18.1 scope.

**Fix (deferred):** When `LLMCheckScheduler` gets a mock-injectable interface, add a coordinator-level integration test that calls `handleHotkeyFired()` through a mock scheduler and asserts `overlayController.suggestions` contains `.llm` entries.
