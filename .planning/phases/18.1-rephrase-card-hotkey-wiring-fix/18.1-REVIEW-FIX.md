### WR-03: Flag-gate regression tests are tautologies

**Files modified:**
- `OpenGram/SuggestionUI/Panels/LLMPanelController.swift`
- `OpenGram/App/CheckCoordinator.swift`
- `OpenGramTests/SuggestionUITests/OverlayControllerFlagOffRegressionTests.swift`

**Commit:** `9213c78`
**Applied fix:**

1. **`LLMPanelShowing` protocol** extracted in `LLMPanelController.swift` — seam for future spy injection. `LLMPanelController` conforms. `CheckCoordinator.llmPanelController` property and init parameter widened to `any LLMPanelShowing`.

2. **`CheckCoordinator.llmPanelGatePasses(cardEnabled:suggestions:) -> Bool`** — static pure function extracted from the inline `if !cardEnabled` gate. Call site updated to use it. Marked `static` (no instance state needed) so tests can invoke it without constructing the full coordinator.

3. **`CheckCoordinatorFlagGatingTests` rewritten** — four behavioral tests replacing the two tautological bool-identity asserts:
   - `flagOn_panelSuppressed`: card=true + suggestions → gate returns false
   - `flagOn_noSuggestions_panelSuppressed`: card=true + empty → gate returns false
   - `flagOff_withSuggestions_panelInvoked`: card=false + suggestions → gate returns true
   - `flagOff_noSuggestions_panelSuppressed`: card=false + empty → gate returns false

   These tests fail if the gate condition is removed or inverted. Full spy-based coordinator test (constructing `CheckCoordinator` with a `LLMPanelShowing` mock) deferred — requires stubbing `StatusBarController` and `LLMCheckScheduler`, per 18.1 CONTEXT.md minimize-diff constraint.

**Test result:** 446 tests passed (all green, including new `CheckCoordinatorFlagGatingTests` suite).
