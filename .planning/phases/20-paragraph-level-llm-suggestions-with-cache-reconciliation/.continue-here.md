## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| GSD phase/plan refs in source | Source comments/filenames that reference "Phase N" or "Plan N" (e.g. `Phase20ParagraphSplitter.swift`, `// Phase 20 tunables`). Landed across 30 files + 2 phase-prefixed filenames before user caught them. | advisory | When writing Swift code in any phase, strip all "Phase N" / "Plan N" prose; use semantic filenames. Preserve requirement IDs (PLL-XX, D-XX, FR-XX, REPH-XX, UAT-XX, WR-XX, SET-XX, NFR-XX). Memory rule: `feedback_no_gsd_refs_in_source.md`. |
| SourceKit stale index noise | After type renames or file adds, SourceKit reports `Cannot find type 'X' in scope` for types that build fine under xcodebuild. | advisory | Ignore SourceKit diagnostics when xcodebuild succeeds; xcodebuild is source of truth per CLAUDE.md. |
| Xcode object-file basename collision | Two Swift files with same basename in same target cause link error (hit during Plan 03 when new ParagraphSplitter.swift collided with ParagraphInfra/ParagraphSplitter.swift). | advisory | When adding a file whose basename already exists in the target, rename one side to a semantic distinguisher \u2014 never use phase prefixes. |
