  <action>
Add a new public method to `OverlayController`:

```swift
func update(suggestions newSuggestions: [Suggestion], context newContext: TextContext)
```

**Implementation:**

1. If overlay window is not visible (not ordered in) OR `self.textContext` is nil: fall through to `show(suggestions:context:)` and return. This handles the first-time case.

2. If `newSuggestions.isEmpty`: call `dismiss()` and return. All errors resolved by user's editing.

3. Compute scalar offsets for the new suggestions (same logic as in `show()` — iterate newSuggestions, compute `unicodeScalars.distance` for each range). Store temporarily as `newOffsets: [(scalarStart: Int, scalarLength: Int)]`.

4. Call `SuggestionDiffEngine.diff(old: self.suggestions, oldOffsets: self.suggestionScalarOffsets, new: newSuggestions, newOffsets: newOffsets)`.

5. If `diff.added.isEmpty && diff.removed.isEmpty`: no-op return (identical suggestion set, no visual change needed). Update `self.textContext = newContext` for freshness but don't touch underlines.

6. Otherwise, build new arrays:
   - Start with empty `survivingSuggestions`, `survivingOffsets`, `survivingEntries` (for underline entries)
   - For each `(oldIndex, newIndex)` in `diff.unchanged`: keep the existing underline entry (from `underlineView`), add the new suggestion and new offset to surviving arrays (use new suggestion data but keep old underline position to avoid unnecessary bounds re-query)
   - For each index in `diff.added`: compute bounds via `BoundsValidator` (same pattern as `show()`), create new `UnderlineEntry`, add to surviving arrays
   - `diff.removed` items are simply not included in surviving arrays (effectively removed)

7. Update internal state: `self.suggestions = survivingSuggestions`, `self.suggestionScalarOffsets = survivingOffsets`, `self.textContext = newContext`

8. Update `underlineView` with the new entries array. Call `underlineView?.updateEntries(survivingEntries)` or equivalent. If `UnderlineView` doesn't have an `updateEntries` method, add one that replaces the entries array and calls `setNeedsDisplay()`.

9. Close any open popover if its suggestion was in `diff.removed`.

**UnderlineView extension (if needed):**
Add `func updateEntries(_ entries: [UnderlineEntry])` that replaces the stored entries and triggers redraw. Check if this method already exists — if UnderlineView stores entries as a property that triggers redraw, just set it.

**Scalar offset computation:**
Extract the scalar-offset-computation logic from `show()` into a private helper `computeScalarOffsets(for suggestions: [Suggestion], in text: String) -> [(scalarStart: Int, scalarLength: Int)]` to avoid duplication between `show()` and `update()`.

   Updated `handleHotkeyFired()`:
```swift
@MainActor
private func handleHotkeyFired() {
    guard let statusBar = statusBarController else { return }

    statusBar.setState(.checking)
    statusBar.updateStatusText("OpenGram: Checking...")

    if let textMonitor {
        textMonitor.forceCheckNow()
    } else {
        // Fallback: direct check (should not happen in normal operation)
        guard let engine = textEngine,
              let harperService = harperService else { return }
        overlayController?.dismiss()
        guard let context = engine.extractText() else {
            statusBar.triggerSilentFail()
            statusBar.updateStatusText("OpenGram: Ready")
            return
        }
        lastExtractedContext = context
        checkTask?.cancel()
        checkTask = Task {
            let suggestions = await harperService.check(text: context.text)
            guard !Task.isCancelled else { return }
            await MainActor.run {
                self.lastSuggestions = suggestions
                if suggestions.isEmpty {
                    statusBar.setState(.done)
                    statusBar.updateStatusText("OpenGram: No issues found")
                } else {
                    statusBar.setState(.done)
                    statusBar.updateStatusText("OpenGram: \(suggestions.count) suggestion(s)")
                    self.overlayController?.show(suggestions: suggestions, context: context)
                }
            }
        }
    }
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Denial of Service | OverlayController.update() bounds queries | mitigate | Only query bounds for genuinely new suggestions (diff.added). Unchanged suggestions reuse existing underline positions. Reduces AX call volume by ~80% on typical re-checks. |
| T-07-08 | Repudiation | Hotkey + TextMonitor double-check race | mitigate | forceCheckNow() cancels pending debounce and in-flight checkTask before starting new check. Only one check in flight at any time (Pitfall 7 from RESEARCH.md). |
| T-07-09 | Tampering | Stale AXUIElement reference after debounce | mitigate | runCheck() re-queries kAXFocusedUIElementAttribute before using stored element reference (Pitfall 5 from RESEARCH.md). |
</threat_model>
