**TextMonitor** is a `@MainActor final class` with these injected dependencies:
- `textEngine: any AXTextEngineProtocol`
- `harperService: any GrammarCheckerProtocol`
- `capabilityCache: any AXCapabilityCacheProtocol`
- `quirksTable: AppQuirksTable` (defaults to `.shared`)
- `watchdog: AXCallWatchdog` (defaults to `.shared`)

**Internal state:**
- `axObserver: AXObserver?`
- `observedElement: AXUIElement?`
- `observedPID: pid_t?`
- `observedBundleID: String?`
- `debounceWork: DispatchWorkItem?`
- `pollTimer: Timer?`
- `lastKnownText: String?`
- `notificationFiredSinceLastPoll: Bool = false`
- `appSwitchObserver: NSObjectProtocol?`
- `appSwitchDebounce: DispatchWorkItem?`
- `checkTask: Task<Void, Never>?`
- `unmanagedContext: Unmanaged<MonitorContext>?`

**Callback types:**
- `var onCheckComplete: ((@MainActor ([Suggestion], TextContext) -> Void))?` — called when a check finishes with results
- `var onDismiss: ((@MainActor () -> Void))?` — called when monitoring pauses (app switch, non-text focus)

**Private nested type:**
```swift
private final class MonitorContext: @unchecked Sendable {
    let handler: @MainActor (String) -> Void
    init(handler: @escaping @MainActor (String) -> Void) {
        self.handler = handler
    }
}
```

**Public API:**
- `func start()` — Subscribe to `NSWorkspace.didActivateApplicationNotification`. Call `installOnFocusedElement()` for the current frontmost app. Per D-07: always-on after launch.
- `func stop()` — Remove workspace observer, uninstall current AXObserver, cancel debounce, stop poll timer, cancel check task.
- `func forceCheckNow()` — Cancel `debounceWork`, cancel `checkTask`, call `runCheck()` immediately. Per D-14: hotkey bypass.

**Private methods:**

`installOnFocusedElement()`:
1. Get frontmost app PID via `NSWorkspace.shared.frontmostApplication?.processIdentifier`
2. Get focused element via `AXUIElementCopyAttributeValue` on `AXUIElementCreateSystemWide()` for `kAXFocusedUIElementAttribute`
3. Call `isTextElement(element)` — if false, call `onDismiss?()` and return
4. Check `watchdog.shouldSkip(for: bundleID)` — if true, return silently (per D-11)
5. Call `installObserver(pid:element:bundleID:)`

`installObserver(pid: pid_t, element: AXUIElement, bundleID: String)`:
1. Call `uninstallCurrentObserver()` first
2. Create `MonitorContext` with handler that routes by notification name:
   - `kAXValueChangedNotification` -> `handleValueChanged()`
   - `kAXFocusedUIElementChangedNotification` -> `installOnFocusedElement()` (field switch within same app, per D-09)
3. Use `Unmanaged.passRetained(context)` pattern from TargetAppObserver
4. `AXObserverCreate(pid, callback, &observer)` with C callback that calls `Task { @MainActor in ctx.handler(name) }`
5. Register `kAXValueChangedNotification` on the text element (per D-04: only value changed, not selection changed)
6. Register `kAXFocusedUIElementChangedNotification` on the app element (for field switches)
7. `CFRunLoopAddSource(RunLoop.main.getCFRunLoop(), AXObserverGetRunLoopSource(observer), .defaultMode)`
8. Store observer, element, PID, bundleID
9. Read initial text value and store as `lastKnownText`
10. Check if app is pre-classified as notification-unreliable via `quirksTable.quirk(for: bundleID)?.notificationUnreliable == true` OR `capabilityCache.isNotificationReliable(bundleID: bundleID) == false`
11. If unreliable or unknown (`isNotificationReliable` returns nil): start poll timer
12. If known reliable: no poll timer needed

`uninstallCurrentObserver()`:
1. If `axObserver` is non-nil: `CFRunLoopRemoveSource`, set to nil
2. Release `unmanagedContext` via `.release()`
3. Stop poll timer
4. Cancel debounce work
5. Clear stored element/PID/bundleID

`handleValueChanged()`:
1. Set `notificationFiredSinceLastPoll = true`
2. Call `scheduleDebounce()`

`scheduleDebounce()`:
1. `debounceWork?.cancel()`
2. Create new `DispatchWorkItem { [weak self] in self?.runCheck() }`
3. `DispatchQueue.main.asyncAfter(deadline: .now() + 0.8, execute: work)` — 800ms per D-05
4. Store as `debounceWork`

`runCheck()`:
1. Guard `observedElement` is non-nil
2. Re-query `kAXFocusedUIElementAttribute` from system-wide element to verify the element is still focused (Pitfall 5 guard)
3. Call `textEngine.extractText()` to get `TextContext`
4. Guard text is non-empty
5. Cancel prior `checkTask`
6. Create new `checkTask = Task { let suggestions = await harperService.check(text: context.text); guard !Task.isCancelled else { return }; await MainActor.run { self.onCheckComplete?(suggestions, context) } }`

`startPollTimer()`:
1. `pollTimer = Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { [weak self] _ in self?.pollForChanges() }` — 1 second per D-03

`stopPollTimer()`:
1. `pollTimer?.invalidate(); pollTimer = nil`

`pollForChanges()`:
1. Guard `observedElement` and `observedBundleID` are non-nil
2. Check `watchdog.shouldSkip(for: bundleID)` — if true, return without resetting `notificationFiredSinceLastPoll` (per Pitfall 3)
3. Read `AXValue` from `observedElement` via `AXUIElementCopyAttributeValue(element, kAXValueAttribute, &value)`
4. Cast to `String`; compare against `lastKnownText`
5. If text changed AND `notificationFiredSinceLastPoll == false`: app missed a notification. Call `capabilityCache.storeNotificationReliability(bundleID: bundleID, reliable: false)` (per D-02 runtime auto-detection)
6. If text changed: update `lastKnownText`, call `scheduleDebounce()`
7. Reset `notificationFiredSinceLastPoll = false`

`handleAppActivation(_ app: NSRunningApplication)`:
1. `appSwitchDebounce?.cancel()`
2. Create new `DispatchWorkItem` with 100ms delay (rapid app-switch debounce):
   - Get `app.bundleIdentifier`; if same as `observedBundleID`, return (no-op)
   - Call `uninstallCurrentObserver()`
   - Call `onDismiss?()` — per D-10: dismiss overlay on app switch
   - Call `installOnFocusedElement()` — attempts to monitor new app's text field

`isTextElement(_ element: AXUIElement) -> Bool`:
1. `AXUIElementCopyAttributeValue(element, kAXRoleAttribute as CFString, &roleRef)`
2. Cast to `String`
3. Return `true` if role is `kAXTextFieldRole`, `kAXTextAreaRole`, or `kAXComboBoxRole`
4. Return `false` otherwise

## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Denial of Service | TextMonitor poll timer | mitigate | Poll timer checks `AXCallWatchdog.shouldSkip()` before every AX call. If an app hangs AX, the watchdog blocklists it for 30s — poll timer respects this and skips silently. |
| T-07-04 | Tampering | AXObserver callback with stale Unmanaged pointer | mitigate | Use `Unmanaged.passRetained` (not passUnretained). Release explicitly in `uninstallCurrentObserver()`. Same proven pattern as TargetAppObserver.swift. |
| T-07-05 | Denial of Service | Rapid app switching floods observer install/uninstall | mitigate | 100ms debounce on `handleAppActivation` prevents registering/unregistering AXObservers for transient intermediate apps during Cmd-Tab. |
| T-07-06 | Information Disclosure | Text read from focused field | accept | Text is read from the user's own focused field via AX API (same as existing hotkey path). No new data exposure beyond what Phase 1 already accesses. Text is never sent externally. |
</threat_model>
