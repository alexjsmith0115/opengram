Build the data layer and diff algorithm that TextMonitor and OverlayController depend on: the SuggestionDiffEngine for flicker-free overlay updates (D-12), AXCapabilityCache notification-reliability storage (D-02), and AppQuirk notificationUnreliable extension (D-02).   <action>
Create `OpenGram/SuggestionUI/SuggestionDiffEngine.swift` with:

1. A `SuggestionKey` struct (internal, Hashable) with fields:
   - `scalarStart: Int`
   - `scalarLength: Int`
   - `original: String`
   - `category: CheckCategory`

2. A `SuggestionDiffResult` struct with:
   - `unchanged: [(oldIndex: Int, newIndex: Int)]` — indices into old/new arrays for matched suggestions
   - `added: [Int]` — indices into the new array for new suggestions
   - `removed: [Int]` — indices into the old array for resolved suggestions

3. A `SuggestionDiffEngine` enum (caseless, namespace) with:
   - `static func diff(old: [Suggestion], oldOffsets: [(scalarStart: Int, scalarLength: Int)], new: [Suggestion], newOffsets: [(scalarStart: Int, scalarLength: Int)]) -> SuggestionDiffResult`
   
   Implementation: Build a `[SuggestionKey: Int]` dictionary from old suggestions (key -> old index). Iterate new suggestions, look up each key in the dictionary. If found, add to `unchanged` and remove from dictionary. After iterating new, remaining dictionary values are `removed` indices. New suggestions not found are `added` indices.

4. A helper `static func key(for suggestion: Suggestion, offset: (scalarStart: Int, scalarLength: Int)) -> SuggestionKey` that builds the key from the parallel arrays (not from the text string — the scalar offsets are already computed by OverlayController.show()).

Per D-12: the diff compares by text range + content, not by UUID (Pitfall 6 from RESEARCH.md — Harper regenerates UUIDs on each check).

Add the new file to the Xcode project (OpenGram target, SuggestionUI group).

Add the test file to the Xcode project (OpenGramTests target).
