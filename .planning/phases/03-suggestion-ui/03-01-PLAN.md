func copyParameterizedAttributeValue(
    _ element: AXUIElement,
    _ attribute: String,
    _ parameter: CFTypeRef

**3. Create OverlayWindow** (OpenGram/SuggestionUI/OverlayWindow.swift):

Per RESEARCH.md Pattern 1 and UI-SPEC Component Inventory. Use `NSPanel` with `.nonactivatingPanel` style mask -- NOT `NSWindow` -- so clicks on underlines do not activate OpenGram and steal focus from the target app.

```swift
import AppKit

@MainActor
final class OverlayWindow: NSPanel {
    /// Called on mouseDown events not consumed by a subview hit-test.
    /// Wired by OverlayController to dismiss on click-outside-underlines.
    var mouseDownHandler: (@MainActor (NSEvent) -> Void)?

    init() {
        super.init(
            contentRect: .zero,
            styleMask: [.nonactivatingPanel],
            backing: .buffered,
            defer: false
        )
        backgroundColor = .clear
        isOpaque = false
        level = .floating
        collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]
        ignoresMouseEvents = false
        hasShadow = false
        hidesOnDeactivate = false      // Overlay must persist when OpenGram is not frontmost
        becomesKeyOnlyIfNeeded = true
    }

    override var canBecomeKey: Bool { false }   // Never steal key from target app
    override var canBecomeMain: Bool { false }

    override func mouseDown(with event: NSEvent) {
        mouseDownHandler?(event)
    }
}
```

NOTE on `canBecomeKey: false`: The original plan had `true` for keyboard events. However, since keyboard navigation is handled via `NSEvent.addGlobalMonitorForEvents(matching: .keyDown)` in Plan 03 (which does not require the window to be key), `canBecomeKey` must be `false` to prevent the overlay from stealing focus. The key monitor approach is used instead of `keyDown(with:)` override because the overlay is non-activating.

NOTE on `.canJoinAllSpaces`: Review LOW concern 9 -- the overlay persists across Space switches. The TargetAppObserver's `kAXApplicationDeactivatedNotification` fires when the user switches Spaces (since the target app deactivates), which triggers dismiss(). Verify this during the manual checkpoint in Plan 03.

**4. Create OverlayController shell** (OpenGram/SuggestionUI/OverlayController.swift):

Create a `@MainActor` class that owns the `OverlayWindow`, stores the current `[Suggestion]` and `TextContext`, and provides the `boundsForRange` method. The controller holds a reference to an `AXAccessor` (injected for testability).

Key methods to implement:
- `boundsForRange(_ suggestion: Suggestion, in text: String, element: AXUIElement) -> CGRect?` -- Converts `suggestion.range` (a `Range<String.Index>`) back to unicode scalar offsets, creates a `CFRange`, wraps it in `AXValueCreate(.cfRange)`, calls `accessor.copyParameterizedAttributeValue(element, kAXBoundsForRangeParameterizedAttribute, axRangeValue)`, unpacks result via `AXValueGetValue(value as! AXValue, .cgRect, &rect)`, returns the CG screen rect.
- `flipCGRect(_ cgRect: CGRect, screenHeight: CGFloat) -> NSRect` -- Converts CG coordinates (y=0 at bottom) to AppKit coordinates: `NSRect(x: cgRect.origin.x, y: screenHeight - cgRect.origin.y - cgRect.height, width: cgRect.width, height: cgRect.height)`.
- `show(suggestions: [Suggestion], context: TextContext)` -- Stores suggestions (capped at `maxDisplayedSuggestions = 50` per T-03-02 to address review MEDIUM concern about main-thread blocking from sequential AX queries), queries bounds for each suggestion, sizes the overlay window to cover the union of all underline rects (plus padding), creates and configures the `UnderlineView` as content view, orders the window front. Silently skips any suggestion whose bounds query returns nil (per UI-SPEC error state). Also populates a parallel `suggestionScalarOffsets: [(scalarStart: Int, scalarLength: Int)]` array by converting each suggestion's `Range<String.Index>` to unicode scalar offsets -- this is used by Plan 03's repositionAfterAccept.
- `dismiss()` -- Orders out the overlay window, removes all state.

Properties:
- `private let accessor: any AXAccessor`
- `private let overlayWindow: OverlayWindow`
- `internal(set) var suggestions: [Suggestion] = []`
- `internal(set) var suggestionScalarOffsets: [(scalarStart: Int, scalarLength: Int)] = []`
- `internal(set) var textContext: TextContext?`
- `private var underlineView: UnderlineView?`
- `private static let maxDisplayedSuggestions = 50`

Constructor: `init(accessor: any AXAccessor = SystemAXAccessor())`.

IMPORTANT (addresses review MEDIUM concern 3): The `show()` method processes up to 50 suggestions sequentially. Each `copyParameterizedAttributeValue` is a synchronous IPC call (~0.5-2ms each). At 50 suggestions this is 25-100ms on the main thread. The 50-cap keeps this below perceptible latency. Document this tradeoff in a code comment. If manual testing reveals noticeable stutter, a future optimization would batch AX calls off the main thread using `Task.detached` (but AX calls must happen on the thread that owns the AXUIElement -- investigate feasibility only if needed).

Do NOT implement popover, keyboard navigation, dismissal triggers, or accept/write-back yet -- those are Plan 02 and Plan 03.

**Data model:**
```swift
struct UnderlineEntry {
    let underlineRect: NSRect   // 2pt tall rect at bottom of glyph bounding box
    let hitRect: NSRect         // underlineRect expanded 6pt above and 6pt below
    let suggestion: Suggestion
}
```

**Properties:**
- `var entries: [UnderlineEntry] = []` -- Set by OverlayController after bounds queries
- `var onClick: ((Suggestion) -> Void)?` -- Callback for click on an underline, wired by OverlayController.

Note: `focusedIndex` from original plan is removed. Phase 6 (D-14, D-15) removed Tab/Enter keyboard navigation in favor of Escape-only via global key monitor. If keyboard focus is re-added later, it would go in a separate plan.

**Key methods:**

`override func draw(_ dirtyRect: NSRect)`:
Per RESEARCH.md Pattern 7 and UI-SPEC:
1. Get `NSGraphicsContext.current?.cgContext`, save state, defer restoreGState.
2. For each entry, draw an `NSBezierPath` line from `(underlineRect.minX, underlineRect.midY)` to `(underlineRect.maxX, underlineRect.midY)` with `lineWidth = 2.0`.
3. Color selection by `suggestion.category`: `.spelling` uses `NSColor.systemRed`, `.grammarPunctuation` uses `NSColor.systemBlue`.
4. Dash pattern by `suggestion.source`: `.harper` is solid (no dash), `.llm` uses `setLineDash([4, 2], count: 2, phase: 0)`. Since Phase 3 only has Harper data, the LLM rendering path exists but will not fire until Phase 4 wires data.
5. Restore graphics state.

`override func hitTest(_ point: NSPoint) -> NSView?`:
Per RESEARCH.md Pitfall 7 -- the overlay window covers a large area but only underline hit rects should accept mouse events. Non-underline clicks must pass through to the target app.
1. Convert `point` from superview coordinates to local coordinates via `superview?.convert(point, to: self) ?? point`.
2. Iterate `entries` and check if the converted point is inside any `hitRect`.
3. If inside a hit rect, return `self` (accept the event).
4. If outside all hit rects, return `nil` (pass-through to target app below the overlay).

This is critical: without this override, the overlay blocks all interaction with the target app while suggestions are showing (RESEARCH.md Pitfall 7, Anti-Patterns).

`override func mouseDown(with event: NSEvent)`:
1. Convert event location to view coordinates via `convert(event.locationInWindow, from: nil)`.
2. Call `suggestionAt(point:)` to find the matching entry.
3. If found, invoke `onClick?(entry.suggestion)`.
4. If not found, pass to `super.mouseDown(with: event)`.

`func suggestionAt(point: NSPoint) -> Suggestion?`:
Iterate `entries`, return first entry where `hitRect.contains(point)`.

`override func resetCursorRects()`:
For each entry, call `addCursorRect(entry.hitRect, cursor: .pointingHand)` per UI-SPEC interaction contract.

**Static helpers (nonisolated for testability without MainActor context):**
- `nonisolated static func colorForCategory(_ category: CheckCategory) -> NSColor` -- returns `.systemRed` for `.spelling`, `.systemBlue` for `.grammarPunctuation`.
- `nonisolated static func isDashedForSource(_ source: SuggestionSource) -> Bool` -- returns `false` for `.harper`, `true` for `.llm`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AX API response -> OverlayController | Untrusted CGRect data from external app's accessibility implementation |
| Mouse events -> UnderlineView | User input events dispatched to underline hit testing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T (Tampering) | boundsForRange | mitigate | AX returns CGRect for char ranges in apps we don't control. Malformed rects (negative dimensions, extreme coordinates, NaN/Infinite) could cause rendering issues. Mitigate: validate all returned rects -- check width/height are finite and positive, clamp to screen bounds before rendering. BoundsValidator handles this validation. |
| T-03-02 | D (Denial of Service) | OverlayController.show | mitigate | Large suggestion count could cause many AX queries, blocking main thread (~0.5-2ms each). Mitigate: cap at 50 suggestions max (`maxDisplayedSuggestions = 50`). At 50 queries, worst case is ~100ms which is below perceptible stutter threshold. |
| T-03-03 | I (Information Disclosure) | OverlayWindow | accept | Overlay window floats above all apps, potentially visible in screenshots/screen recordings. Accept: this is inherent to the feature design -- user invokes it intentionally. |
</threat_model>
