    **AXCallWatchdog.swift** (new file at OpenGram/SuggestionUI/AXCallWatchdog.swift):
    - `final class AXCallWatchdog` -- singleton via `static let shared`
    - Thread safety via `NSLock` (not DispatchQueue -- NSLock is lower overhead for frequent lock/unlock)
    - `struct ActiveCall` value type: `bundleID: String`, `startTime: Date`, `attribute: String`
    - `private var activeCall: ActiveCall?`
    - `private var blocklist: [String: Date] = [:]` -- maps bundleID to expiration timestamp
    - `private let hangThreshold: TimeInterval = 0.8`
    - `private let blocklistDuration: TimeInterval = 30.0`
    - `func shouldSkip(for bundleID: String) -> Bool`: lock, check blocklist (remove if expired, return true if not expired), check activeCall busy guard (skip if another call in flight < 1.2s), unlock, return false
    - `func beginCall(bundleID: String, attribute: String)`: lock, set activeCall, unlock
    - `func endCall()`: lock, clear activeCall, unlock
    - `private var timer: DispatchSourceTimer?` -- background timer on `.global(qos: .utility)` firing every 0.1s
    - Timer handler: lock, check if activeCall exists and `Date().timeIntervalSince(activeCall.startTime) > hangThreshold`, if so add `activeCall.bundleID` to blocklist with expiration `Date() + blocklistDuration` and nil out activeCall, unlock
    - Timer uses `[weak self]` in handler closure (singleton lives for app lifetime but good practice)
    - `init()` starts the timer. No public `stop()` needed (singleton lifetime).
    - For testability: add an internal initializer `init(hangThreshold: TimeInterval, blocklistDuration: TimeInterval)` so tests can use shorter durations. The `shared` singleton uses the production defaults.

    **BoundsValidator.swift** (new file at OpenGram/SuggestionUI/BoundsValidator.swift):
    - `struct BoundsValidator` (not a class -- stateless, all methods take parameters)
    - Primary method: `func validatedBoundsForRange(_ suggestion: Suggestion, in text: String, element: AXUIElement, bundleID: String, accessor: any AXAccessor) -> [NSRect]?`
      1. Check `AXCallWatchdog.shared.shouldSkip(for: bundleID)` -- return nil if true
      2. Compute CFRange from suggestion using scalar offsets (same logic as current `cfRangeFor`)
      3. Create AXValue, call `AXCallWatchdog.shared.beginCall(...)`, make the `copyParameterizedAttributeValue` call, call `AXCallWatchdog.shared.endCall()`
      4. Guard: `error == .success`, ref is non-nil
      5. Guard: `CFGetTypeID(ref) == AXValueGetTypeID()` before casting (per Pitfall 2 -- NO force cast)
      6. Unpack to CGRect
      7. Validate: `width >= 2 && height >= 2 && width < 800 && height < 200 && !origin.x.isNaN && !origin.y.isNaN && !origin.x.isInfinite && !origin.y.isInfinite`
      8. Apply quirk coordinate offsets: `let quirk = AppQuirksTable.shared.quirk(for: bundleID)`, apply `coordinateOffsetX/Y` if present
      9. Check for multi-line: if `rect.height > typicalLineHeight * 1.5` (where typicalLineHeight is obtained by querying bounds for a single character at the start of the range), call `splitMultiLine(...)`
      10. If not multi-line: flip coordinates using `primaryScreenHeight` (use `NSScreen.screens.first { $0.frame.origin == .zero }`, NOT `NSScreen.main` per Pitfall 1), return `[flippedRect]`
    - `private func splitMultiLine(element: AXUIElement, range: CFRange, overallRect: CGRect, accessor: any AXAccessor, bundleID: String) -> [NSRect]?`
      1. Query `kAXLineForIndexParameterizedAttribute` for range.location and range.location + range.length - 1 to get start/end line numbers
      2. For each line number: query `kAXRangeForLineParameterizedAttribute` to get that line's character range
      3. Intersect error range with each line range
      4. Query `kAXBoundsForRangeParameterizedAttribute` for each per-line intersection range
      5. Validate each per-line rect (same min/max checks)
      6. Flip each to AppKit coordinates
      7. Fallback: if `AXLineForIndex` fails, sample characters at 5-char intervals across the range, query single-char bounds, group by Y-coordinate (Y values within lineHeight/2 of each other belong to the same line), then query bounds for each Y-group's character range
    - `private func cfRangeFor(_ suggestion: Suggestion, in text: String) -> CFRange` -- extracted from OverlayController (same logic: unicodeScalars distance)
    - `private func flipCGRect(_ cgRect: CGRect) -> NSRect` -- uses primary screen height, not NSScreen.main
    - Constants: `static let minimumBoundsWidth: CGFloat = 2`, `minimumBoundsHeight: CGFloat = 2`, `maximumBoundsWidth: CGFloat = 800`, `maximumBoundsHeight: CGFloat = 200`
    - All methods that call AX APIs wrap with watchdog beginCall/endCall

    **AppQuirksTable.swift** (new file at OpenGram/AppQuirks/AppQuirksTable.swift):
    - `struct AppQuirk: Codable` with optional fields: `coordinateOffsetX: CGFloat?`, `coordinateOffsetY: CGFloat?`, `lineHeightFactor: CGFloat?`, `boundsStrategy: String?` (values: "rangeBounds" default, "skipMultiLine")
    - `final class AppQuirksTable` -- singleton via `static let shared`
