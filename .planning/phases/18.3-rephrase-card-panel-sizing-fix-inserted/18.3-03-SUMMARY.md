## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed detached NSHostingView fittingSize measurement**
- **Found during:** Task 2 (test failures showed fittingSize always returned 260 = idealHeight)
- **Issue:** `RephraseCardPanelController.show()` called `layoutSubtreeIfNeeded()` on a hosting view not yet attached to any panel. SwiftUI has no window context → `fittingSize` collapses to `idealHeight` (260pt) regardless of content length. The overflow branch could never fire.
- **Fix:** Set `newPanel.contentView = hosting` before `layoutSubtreeIfNeeded()` (matching `LLMPanelController.swift:64`). Added `testHookFittingSize` seam for remaining headless limitation.
- **Files modified:** `OpenGram/SuggestionUI/Panels/RephraseCardPanelController.swift`
- **Commit:** `fa11172`

**2. [Rule 2 - Missing critical functionality] Added testHookFittingSize seam**
- **Found during:** Task 2 (even after production fix, headless NSHostingView still returns idealHeight — platform limitation)
- **Issue:** Without a real display, `NSHostingView.fittingSize` never exceeds `idealHeight`. The overflow branch (NSScrollView wrapper) was unreachable in any headless test.
- **Fix:** `internal var testHookFittingSize: NSSize?` on controller; tests inject oversized value before `show()`; production leaves nil.
- **Files modified:** `OpenGram/SuggestionUI/Panels/RephraseCardPanelController.swift`
- **Commit:** `fa11172`

**3. [Rule 2 - Missing import] Added import SwiftUI to test file**
- **Found during:** Task 2 planning
- **Issue:** `NSHostingView<RephraseCardView>` type check requires `SwiftUI` visible in test file.
- **Fix:** Added `import SwiftUI` to `RephraseCardPanelControllerTests.swift`.
- **Commit:** `fa11172`
