## Warnings

### WR-01: `PanelPositioner.capHeight` returns negative height when `margin > visibleFrame.height`

**File:** `OpenGram/SuggestionUI/Panels/PanelPositioner.swift:72-73`
**Issue:** `let maxHeight = visibleFrame.height - margin` is unguarded. If `margin` exceeds `visibleFrame.height` (theoretically possible if the screen reports an unexpectedly small visibleFrame — e.g., during Mission Control or in certain test environments), `maxHeight` is negative and `min(size.height, maxHeight)` returns a negative height. `NSPanel.setContentSize` with negative dimensions produces undefined AppKit behavior.
**Fix:**
```swift
static func capHeight(
    _ size: NSSize,
    visibleFrame: NSRect,
    margin: CGFloat
) -> NSSize {
    let maxHeight = max(visibleFrame.height - margin, 1)   // guard: never negative
    return NSSize(width: size.width, height: min(size.height, maxHeight))
}
```

### WR-02: `testHookFittingSize` is a production-visible mutable seam with no write-access guard

**File:** `OpenGram/SuggestionUI/Panels/RephraseCardPanelController.swift:32`
**Issue:** `internal var testHookFittingSize: NSSize?` is settable by any code in the module (not just `@testable` test targets). `internal` in a single-target app with no module boundary between app code and lib code means it is reachable from `OverlayController`, `AppDelegate`, etc. with no compile-time friction. The `testHookPanel` accessor is read-only by design; `testHookFittingSize` is read-write with no analogous guard. If any app-side code inadvertently sets it (e.g., copy-paste or refactor), the overflow branch fires permanently with the injected size and panels show incorrect heights silently.

A minimal mitigation is a `#if DEBUG` guard or a comment co-located with the property that makes the test-only contract explicit and enforceable by tooling.
**Fix:**
```swift
/// Test-only seam. MUST remain nil in production — setting this in non-test code
/// causes incorrect panel sizing. Wrapped in #if DEBUG to prevent accidental production use.
#if DEBUG
internal var testHookFittingSize: NSSize?
#endif
```
Then guard the usage site:
```swift
#if DEBUG
let fitting = testHookFittingSize ?? hosting.fittingSize
#else
let fitting = hosting.fittingSize
#endif
```

### IN-01: `verticalSafeMargin` magic value not documented at call site

**File:** `OpenGram/SuggestionUI/Panels/RephraseCardPanelController.swift:22`
**Issue:** `private static let verticalSafeMargin: CGFloat = 40` has no comment explaining what the 40pt buys (menu bar height ~24pt + a small visual buffer). The constant name is good, but the rationale for the specific value is absent. Future maintainers may adjust it without understanding the constraint.
**Fix:** Add one-line rationale comment:
```swift
// 40pt = ~24pt menu bar + ~16pt visual breathing room so panel never clips behind menu bar.
private static let verticalSafeMargin: CGFloat = 40
```
