- No maxHeight added to outer frame — height cap is controller responsibility (D-02/D-04).

## Deviations from Plan

## Issues Encountered

- Initial RED test (`fittingSize > 260`) did not fail as expected because `idealHeight: 260` on the outer `.frame` constrains `NSHostingView.fittingSize` to 260pt even without the ScrollView, regardless of initial hosting frame size. Root cause: this is correct SwiftUI behavior — `idealHeight` propagates through `NSHostingView.fittingSize`. The controller-layer fix (Plan 02) resolves the real sizing problem by sizing the panel to a larger contentSize first. Redesigned test to cover multi-segment rendering correctness instead.
