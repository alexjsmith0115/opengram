## Deviations / scope creep resolved during checkpoint

Three unrelated defects surfaced while wiring up manual validation.
Closing them was prerequisite for the card to render at all; all fixed
inline rather than deferring:

1. **`llmIncrementalCheckingEnabled` feature flag removed.** Legacy
   `checkFullText` path created Suggestion with `paragraphHash: nil`,
   making LLM suggestions invisible to the rephrase-card dispatch filter
   (which requires `paragraphHash == hash`). Flag violated CLAUDE.md's
   "No feature flags, no gated rollouts" principle. Deleted flag,
   protocol member, legacy path, and 11 flag-off tests. Per-paragraph
   `rebase()` path is now the only path.

2. **Legacy `LLMPanelController` chain deleted.** The old 3-section
   Clarity/Tone/Rephrase side panel was still wired via
   `TextMonitor.onLLMBatch` → `CheckOrchestrator.runCheck` LLM block →
   `CheckCoordinator.handleLLMBatch` → `showLLMPanel`. Fired duplicate
   UI alongside the new rephrase card on every LLM response. Phase 18.2
   removed the feature flag but missed this code path. Deleted
   `LLMPanelController.swift`, `LLMSuggestionPanel.swift`,
   `TwoPhaseCheckFlowTests.swift`, plus all `onLLMBatch` wiring and
   `handleLLMBatch` / `showLLMPanel` / `applyLLMSuggestion` helpers.

3. **fittingSize measurement order bug in Plan 02.** Initial
   implementation called `hosting.fittingSize` before attaching
   `hosting` to `panel.contentView`. Detached `NSHostingView` returns
   the `idealHeight` value (260) regardless of real content. Moved
   `contentView = hosting` + `layoutSubtreeIfNeeded()` before the
   `fittingSize` read (matches LLMPanelController legacy pattern).
