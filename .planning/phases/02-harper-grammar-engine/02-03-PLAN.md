**Replace the handleHotkeyFired() method** with this implementation (per D-09, D-11, and research AppDelegate integration pattern):

```swift
@MainActor
private func handleHotkeyFired() {
    guard let statusBar = statusBarController,
          let engine = textEngine,
          let harperService = harperService else { return }

    statusBar.setState(.checking)
    statusBar.updateStatusText("OpenGram: Checking...")

    guard let context = engine.extractText() else {
        statusBar.triggerSilentFail()
        statusBar.updateStatusText("OpenGram: Ready")
        return
    }

    lastExtractedContext = context

    Task {
        let suggestions = await harperService.check(text: context.text)
        await MainActor.run {
            self.lastSuggestions = suggestions
            print("[OpenGram] Harper found \(suggestions.count) suggestion(s)")
            statusBar.setState(.done)
            statusBar.updateStatusText("OpenGram: Ready")
        }
    }
}
```

Key changes from Phase 1 version:
- Adds `harperService` to the guard at the top
- Wraps the Harper check in `Task { }` (per D-09: background execution for potentially long text)
- `statusBar.setState(.checking)` is called on MainActor BEFORE the Task (user sees immediate feedback)
- `statusBar.setState(.done)` is called on MainActor AFTER the check completes (inside `await MainActor.run {}`)
- `lastSuggestions` stored for Phase 3 consumption
- Per D-11: console print with suggestion count only (no user text content logged)
- `lastExtractedContext` is set before the Task (not inside it) so it's available immediately

| HarperService -> Console log | Debug output must not leak user text content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02-08 | Information Disclosure | AppDelegate console logging | mitigate | Per D-11, log only suggestion count (`suggestions.count`), never the user's text content or suggestion details. Phase 3 removes debug logging entirely. |
| T-02-09 | Denial of Service | Large text input | mitigate | Per D-09, Harper runs inside `Task {}` on the actor's background executor. `statusBar.setState(.checking)` gives user feedback. No text length limit, but the async boundary prevents UI freeze. |
