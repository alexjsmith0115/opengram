Run: `brew install rustup` (Homebrew has rustup as keg-only formula).
Then: `eval "$(brew shellenv)"` to update PATH.
Then: `rustup default stable` to install the stable Rust toolchain.
Then: `rustup target add aarch64-apple-darwin x86_64-apple-darwin` for both macOS architectures.
Verify: `rustc --version` outputs 1.85+ and `cargo --version` outputs a matching version.

**Step 2: Create harper-bridge/Cargo.toml** with these exact contents:
```toml
[package]
name = "harper-bridge"
version = "0.1.0"
edition = "2021"

[lib]
name = "harper_bridge"
crate-type = ["staticlib", "cdylib"]

[dependencies]
harper-core = "=2.0.0"
uniffi = { version = "=0.31.0", features = ["scaffolding"] }

[[bin]]
name = "uniffi-bindgen-swift"
path = "uniffi-bindgen-swift.rs"
```

Note: Both harper-core and uniffi are pinned with `=` exact version. This is critical -- UniFFI version mismatch causes runtime panics (Pitfall 1 from research).

**Step 4: Create harper-bridge/src/lib.rs** implementing the bridge. This is the core deliverable. Follow the reference implementation from 02-RESEARCH.md Pattern 1, with these exact requirements:

- `uniffi::setup_scaffolding!()` at top
- `SuggestionCategory` enum with two variants: `Spelling` and `GrammarPunctuation` (per D-03: all non-Spelling LintKind variants map to GrammarPunctuation)
- `GrammarSuggestion` record with fields: `start_char: u32`, `end_char: u32`, `message: String`, `primary_replacement: Option<String>`, `all_replacements: Vec<String>`, `category: SuggestionCategory`, `priority: u8` (per D-01: carry all replacements, primary shown prominently)
- `HarperChecker` object with `#[uniffi::constructor] fn new(dialect_abbr: String, user_words: Vec<String>) -> Self`
  - Uses `FstDictionary::curated()` for base dictionary
  - Creates `MutableDictionary`, loads user_words into it
  - Creates `MergedDictionary` layering user dict over base
  - Creates `LintGroup::new_curated(merged_dict, dialect)` (per GRAM-08)
  - `parse_dialect()` helper: "GB"->British, "CA"->Canadian, "AU"->Australian, "IN"->Indian, default->American
- `fn check(&mut self, text: String) -> Vec<GrammarSuggestion>` method:
  - Creates `Document::new_curated(&text, &PlainEnglish)` (per D-10: trust PlainEnglish parser)
  - Calls `self.linter.lint(&document)` to get Vec<Lint>
  - Maps each Lint to GrammarSuggestion, handling all three Suggestion variants: `ReplaceWith`, `InsertAfter`, `Remove` (Pitfall from research: do NOT ignore InsertAfter)
  - Category mapping: `LintKind::Spelling` -> `SuggestionCategory::Spelling`, everything else -> `SuggestionCategory::GrammarPunctuation`
  - Span offsets are char indices (NOT byte offsets -- this is the v2.0.0 correction from research)
- `fn add_to_dictionary(&mut self, word: String) -> Vec<String>` (per D-05: expose API for Phase 3/5 UI)
- `fn set_rule_enabled(&mut self, rule_key: String, enabled: bool)` (per GRAM-09)

**IMPORTANT API notes from research:**
- `LintGroup` is `!Send` -- the Swift actor wrapper (Plan 02) handles thread safety
- `FstDictionary::curated()` returns `Arc<FstDictionary>` -- do NOT create a new one per check
- The exact generic signature for the linter field may need adjustment during compilation. If `MergedDictionary<Arc<FstDictionary>, MutableDictionary>` does not compile, try wrapping differently per compiler guidance. The research flagged this as Assumption A1.

**Step 1: Create build-harper.sh** at the repo root with these exact contents. The script must:

- Start with `#!/bin/bash` and `set -euo pipefail`
- Export PATH to include both `$HOME/.cargo/bin` and `$(brew --prefix rustup)/bin` (Pitfall 6: Xcode Run Script won't find cargo otherwise)
- Set REPO_ROOT, CRATE_DIR, FRAMEWORK_NAME="HarperBridge", OUT_DIR, SWIFT_OUT variables
- `cd "$CRATE_DIR"`
- Build for aarch64-apple-darwin: `cargo build --target aarch64-apple-darwin --release`
- Build for x86_64-apple-darwin: `cargo build --target x86_64-apple-darwin --release`
- Generate Swift bindings using the project-local uniffi-bindgen-swift binary:
  ```
  mkdir -p bindings
  cargo run -p uniffi-bindgen-swift -- \
    target/aarch64-apple-darwin/release/libharper_bridge.a \
    bindings --swift-sources
  cargo run -p uniffi-bindgen-swift -- \
    target/aarch64-apple-darwin/release/libharper_bridge.a \
    bindings/include --headers
  cargo run -p uniffi-bindgen-swift -- \
    target/aarch64-apple-darwin/release/libharper_bridge.a \
    bindings/Modules --modulemap --xcframework
  ```
- Create universal binary: `lipo -create target/aarch64-apple-darwin/release/libharper_bridge.a target/x86_64-apple-darwin/release/libharper_bridge.a -output target/universal/release/libharper_bridge.a`
- Package xcframework: `xcodebuild -create-xcframework -library target/universal/release/libharper_bridge.a -headers bindings/include -output "$OUT_DIR"`
- Copy Swift bindings: `cp bindings/*.swift "$SWIFT_OUT/"`

**IMPORTANT:** The exact uniffi-bindgen-swift invocation flags may differ from the research template. The research notes this as Assumption A2. If `cargo run -p uniffi-bindgen-swift` fails or the flags are wrong, check `cargo run -p uniffi-bindgen-swift -- --help` for the correct syntax and adjust.
