**Part B: Create OpenGram/CheckEngine/Suggestion.swift** with the following contents:

1. **String extension for Unicode scalar offset conversion:**
   - `func indexFromCharOffset(_ offset: Int) -> String.Index?` -- converts a Harper char index (Unicode scalar position) to a Swift String.Index by advancing through `self.unicodeScalars`. Return nil if offset < 0 or offset > scalars.count.
   - `func rangeFromCharOffsets(start: Int, end: Int) -> Range<String.Index>?` -- converts a Harper Span's start/end char indices to a Swift Range<String.Index>. Return nil if either index is invalid or start > end.
   - **MANDATORY grapheme cluster boundary validation (Pitfall 3 from RESEARCH.md):** After computing the `Range<String.Index>` from the unicodeScalars view, validate that both the lower and upper bounds land on grapheme cluster boundaries. Do this by round-tripping each bound through `self.rangeOfComposedCharacterSequence(at: index).lowerBound`. If the round-tripped lower bound does not equal the original, snap to the grapheme boundary. Specifically:
     ```swift
     func rangeFromCharOffsets(start: Int, end: Int) -> Range<String.Index>? {
         guard let s = indexFromCharOffset(start),
               let e = indexFromCharOffset(end),
               s <= e else { return nil }
         // Snap to grapheme cluster boundaries to prevent mid-cluster slicing
         let safeLower = self.rangeOfComposedCharacterSequence(at: s).lowerBound
         let safeUpper = self.rangeOfComposedCharacterSequence(at: e).lowerBound
         guard safeLower <= safeUpper else { return nil }
         return safeLower..<safeUpper
     }
     ```
     This prevents a runtime crash or garbled output if a Harper span endpoint lands at a Unicode scalar that is mid-grapheme-cluster (e.g., inside a composed emoji like family emoji using ZWJ sequences). The snap rounds outward to the enclosing grapheme boundary.

   **Critical: Harper char == Rust char == Unicode scalar value.** Swift's `String.unicodeScalars` view is a 1:1 match. Do NOT use UTF-8 byte offsets (the integration research doc's `indexFromByteOffset` is wrong for v2.0.0).

**Step 3: Create OpenGram/CheckEngine/HarperService.swift.**

Implement as a Swift `actor` conforming to `HarperCheckerProtocol`:
```swift
actor HarperService: HarperCheckerProtocol {
    private var checker: HarperChecker  // UniFFI-generated class from HarperBridge.swift
    private let dictionaryStore: any DictionaryStoreProtocol

    init(dictionaryStore: any DictionaryStoreProtocol, dialect: String) {
        let words = dictionaryStore.loadWords()
        self.checker = HarperChecker(dialectAbbr: dialect, userWords: words)
        self.dictionaryStore = dictionaryStore
    }

    func check(text: String) -> [Suggestion] {
        let raw = checker.check(text: text)
        return raw.compactMap { Suggestion(from: $0, in: text) }
    }

    func addToDictionary(word: String) {
        let updatedWords = checker.addToDictionary(word: word)
        dictionaryStore.saveWords(updatedWords)
    }

    func setRuleEnabled(key: String, enabled: Bool) {
        checker.setRuleEnabled(ruleKey: key, enabled: enabled)
    }
}
```

**Critical notes:**
- `HarperService` is an `actor`, NOT a class. This provides the single-threaded execution context that `LintGroup` (`!Send` in Rust) requires (Pitfall 2).
- Do NOT mark `HarperChecker` as `@Sendable` or `nonisolated(unsafe)`. Keep it isolated within the actor.
- The `check()` method does NOT need the `async` keyword in the body -- the actor boundary provides the async contract. The protocol declares it as `async` so callers use `await`.
- `dictionaryStore` is typed as `any DictionaryStoreProtocol` for testability (matching Phase 1 DI pattern).
